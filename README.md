# セットプレイ・フレーム消費検索アプリ

格闘ゲームのセットプレイ検証を補助するローカルブラウザ完結型 Web アプリです。  
コンボ後・投げ後などの有利フレームと目標フレームを入力すると、登録済み技データから必要消費フレームを満たす技の組み合わせを検索します。

---

## 機能

- ゲーム / 操作タイプ / キャラクター別に技データを管理
- フレーム消費検索（最大技数・同技複数使用・待ちF許可 等）
- 重ねモード（持続を重ねる攻撃の必要消費F自動計算）
- セットプレイ登録・一覧・詳細編集
- 始動プリセット（前投げ +71F など有利Fに名前をつけて管理）
- プリセット設定（ゲーム別の基本技表・検索デフォルトルール・タグ管理）
- JSON エクスポート / インポート / 全データ初期化
- データはすべてブラウザの IndexedDB に保存（サーバー不要・ログイン不要）

---

## 技術スタック

| 項目 | 内容 |
|------|------|
| フレームワーク | Vite + React + TypeScript |
| データ保存 | Dexie（IndexedDB ラッパー） |
| バックエンド | なし |
| 認証 | なし |
| 主な依存 | dexie のみ |

## ローカル開発

### 前提条件

- Node.js 18 以上

### セットアップ

```bash
# 1. 依存パッケージをインストール
npm install

# 2. 開発サーバーを起動（ホットリロードあり）
npm run dev
```

ブラウザで `http://localhost:5173` を開くと動作確認できます。

### 開発チェック

```bash
npm run check
npm run build
```

詳細は [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md) を参照してください。

## ビルド

```bash
npm run build
```

`dist/` フォルダが生成されます。これが Netlify へアップロードする成果物です。

> `dist/` に含まれるもの：
> - `index.html`
> - `assets/`（ハッシュ付き JS・CSS）
> - `_redirects`（Netlify SPA フォールバック設定）

---

## Netlify 手動デプロイ手順

### 初回デプロイ

1. ビルドを実行する

   ```bash
   npm run build
   ```

2. [Netlify](https://app.netlify.com/) にログインする

3. **Sites** 画面の下部にある  
   **"Want to deploy a new site without connecting to Git? Deploy manually."** をクリックする

4. 表示されたドロップゾーンに **`dist` フォルダをドラッグ＆ドロップ** する

5. デプロイ完了後、発行された URL（例：`https://xxxxxxxx.netlify.app`）でアクセスできる

### アプリ更新時の手順

1. コードを修正する

2. ビルドを実行する

   ```bash
   npm run build
   ```

3. Netlify の当該サイトのダッシュボードを開く

4. **「Deploys」** タブ → **「Deploy manually」** エリアに  
   新しい **`dist` フォルダ** をドラッグ＆ドロップする

5. デプロイ完了後、同じ URL で最新版が反映される

> **キャッシュについて**  
> `index.html` は `no-cache` 設定のため、ユーザーがリロードすると自動的に最新版が読み込まれます。  
> `assets/` 以下はビルドごとにファイル名が変わるため、古いキャッシュが残っても問題ありません。

## データの扱い

### IndexedDB スキーマバージョン管理

Dexie の `version()` でバージョン管理しています。スキーマ変更時はバージョンを上げることで既存データを保持したまま安全に移行できます。

| DB バージョン | 追加テーブル |
|-------------|------------|
| 1 | characters, moves, setplays |
| 2 | presets |
| 3 | starters |

### バックアップ・復元

- アプリ内「**💾 データ管理**」タブから全データを JSON エクスポートできます
- ブラウザのキャッシュ削除・PC 移行の前には必ずエクスポートしてください
- エクスポートした JSON は同タブの「インポート」から復元できます

## バージョン管理

アプリ画面右上にバージョン番号が常時表示されます。  
更新時は以下のファイルのバージョンを上げてください。

```ts
// src/version.ts
export const APP_VERSION = '1.0.0';      // ← 更新ごとに変更
export const DB_SCHEMA_VERSION = 3;      // DB スキーマを変えたら増やす
export const DATA_FORMAT_VERSION = 2;    // エクスポート形式を変えたら増やす
```

```json
// package.json
{
  "version": "1.0.0"
}
```
